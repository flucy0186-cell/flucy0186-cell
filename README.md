## 我的产品思考方式

```
问题     →  一线真实摩擦是什么？用户是谁、不是什么？
   ↓
决策     →  价值与风险/监管目标是否冲突？该不该继续这个方向？
   ↓
方案     →  MVP 边界：辅助什么、不替代什么、谁确认、谁负责？
   ↓
事实依据  →  决策与产出是否可追溯、可审核、可复盘？
```

**三个原则**

- **合规场景**：先问「该不该自动化」，再问「怎么自动化」
- **知识工作流**：先建立可检索、可去重的索引层，再支撑案例与对外产出
- **AI 协作生产**：先固定可验证事实，再生成案例与对外叙事
